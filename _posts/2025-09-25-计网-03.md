<h2>第3章 数据链路层 </h2>

<h3>一、数据链路层作用</h3>
<ul>
  <li>提供<b>可靠的点到点传输</b></li>
  <li>实现<b>差错控制、流量控制</b></li>
  <li>完成<b>帧封装</b></li>
</ul>

<h3>二、基本概念</h3>
<ul>
  <li><b>节点：</b>主机、交换机、路由器</li>
  <li><b>链路：</b>物理连接</li>
  <li><b>数据链路：</b>链路 + 协议</li>
</ul>

<ul>
  <li><b>两种链路：</b>
    <ul>
      <li>点对点链路</li>
      <li>广播链路</li>
    </ul>
  </li>
</ul>

<h3>三、三大核心问题</h3>
<ul>
  <li>封装成帧</li>
  <li>透明传输</li>
  <li>差错控制</li>
</ul>

<h3>四、封装成帧</h3>
<ul>
  <li><b>帧 = 头部 + 数据 + 尾部</b></li>
  <li><b>MTU：</b>最大传输单元</li>
  <li>帧定界：SOH / EOT 或标志位</li>
</ul>

<h3>五、透明传输</h3>
<ul>
  <li>问题：数据中出现“控制字符”</li>
  <li><b>解决方法：</b>
    <ul>
      <li>字符填充（ESC）</li>
      <li>比特填充（PPP用）</li>
    </ul>
  </li>
</ul>

<h3>六、差错控制</h3>
<ul>
  <li><b>误码率 BER</b></li>
  <li>错误类型：
    <ul>
      <li>单比特错</li>
      <li>突发错</li>
    </ul>
  </li>
</ul>

<ul>
  <li><b>两类编码：</b>
    <ul>
      <li>检错码（CRC）</li>
      <li>纠错码（海明码）</li>
    </ul>
  </li>
</ul>

<h3>七、CRC</h3>
<ul>
  <li>在数据后加<b>FCS</b></li>
  <li>接收端重新计算 → 判断是否出错</li>
</ul>

<h3>八、PPP协议</h3>

<h4>1. 特点</h4>
<ul>
  <li>点对点协议</li>
  <li>面向字节</li>
</ul>

<h4>2. 组成</h4>
<ul>
  <li>LCP（链路控制）</li>
  <li>NCP（网络控制）</li>
</ul>

<h4>3. 帧格式</h4>
<ul>
  <li>Flag(7E) + Address(FF) + Control(03) + Protocol + Data + FCS + Flag</li>
</ul>

<h4>4. 透明传输</h4>
<ul>
  <li>比特填充（连续5个1插0）</li>
  <li>字节填充（0x7D转义）</li>
</ul>

<h3>九、数据链路层子层</h3>
<ul>
  <li>LLC：逻辑链路控制（已弱化）</li>
  <li>MAC：介质访问控制（重点）</li>
</ul>

<h3>十、MAC协议</h3>

<h4>1. 信道划分</h4>
<ul>
  <li>TDMA（时分）</li>
  <li>FDMA（频分）</li>
  <li>CDMA（码分）</li>
</ul>

<h4>2. 随机接入</h4>
<ul>
  <li>ALOHA</li>
  <li>CSMA/CD（有线）</li>
  <li>CSMA/CA（无线）</li>
</ul>

<h4>3. 轮流协议</h4>
<ul>
  <li>轮询</li>
  <li>令牌传递</li>
</ul>

<h3>十一、CSMA/CD</h3>
<ul>
  <li>先听后发</li>
  <li>边发边检测冲突</li>
  <li>冲突后随机退避（指数退避）</li>
</ul>

<h3>十二、MAC地址</h3>
<ul>
  <li>48位（6字节）</li>
  <li>全球唯一</li>
</ul>

<ul>
  <li>类型：
    <ul>
      <li>单播</li>
      <li>多播</li>
      <li>广播 FF:FF:FF:FF:FF:FF</li>
    </ul>
  </li>
</ul>

<h3>十三、以太网</h3>

<h4>1. 帧结构</h4>
<ul>
  <li>Preamble + Dest + Src + Type + Data + CRC</li>
</ul>

<h4>2. 长度</h4>
<ul>
  <li>最小：64B</li>
  <li>最大：1518B</li>
</ul>

<h4>3. 冲突域与广播域</h4>
<ul>
  <li>Hub：不隔离</li>
  <li>Switch：隔离冲突域</li>
  <li>Router：隔离广播域</li>
</ul>

<h3>十四、交换机</h3>
<ul>
  <li>基于MAC地址转发</li>
  <li>维护MAC表（自学习）</li>
</ul>

<h3>十五、VLAN</h3>
<ul>
  <li>逻辑划分广播域</li>
</ul>

<ul>
  <li><b>IEEE 802.1Q帧：</b>
    <ul>
      <li>在MAC帧中插入<b>4字节标签</b></li>
      <li>包含 VLAN ID</li>
    </ul>
  </li>

  <li><b>交换机转发：</b>
    <ul>
      <li>同 VLAN 内转发</li>
      <li>不同 VLAN 需三层设备</li>
    </ul>
  </li>
</ul>

<h3>十六、PPPoE</h3>
<ul>
  <li>PPP over Ethernet</li>
  <li>用于宽带拨号（ADSL）</li>
  <li>在以太网上封装PPP帧</li>
</ul>
