<h2>第5章 传输层 </h2>

<h3>一、传输层概述</h3>

<h4>1. 传输层作用</h4>

<ul>
  <li>为应用进程提供端到端通信</li>
  <li>实现复用与分用</li>
  <li>实现差错控制、流量控制</li>
</ul>

<h3>四、UDP协议</h3>

<h4>1. UDP特点</h4>

<ul>
  <li>无连接</li>
  <li>不可靠</li>
  <li>面向报文</li>
  <li>无流量控制</li>
  <li>无拥塞控制</li>

  <li>直播</li>
  <li>语音</li>
  <li>视频</li>
  <li>游戏</li>
</ul>

<h3>六、TCP协议</h3>

<h3>八、TCP连接管理</h3>

<h4>1. 三次握手</h4>

<h5>过程：</h5>

<ol>
  <li>客户端发送 SYN</li>
  <li>服务器回复 SYN + ACK</li>
  <li>客户端回复 ACK</li>
</ol>

<h4>2. 为什么三次？</h4>

<ul>
  <li>防止失效连接请求</li>
  <li>确认双方收发能力正常</li>
</ul>

<hr>

<h3>九、TCP释放连接</h3>

<h4>四次挥手</h4>

<ol>
  <li>FIN</li>
  <li>ACK</li>
  <li>FIN</li>
  <li>ACK</li>
</ol>

<h4>TIME_WAIT（高频）</h4>

<ul>
  <li>等待2MSL</li>
</ul>

<h4>作用</h4>

<ul>
  <li>保证最后ACK到达</li>
  <li>防止旧连接报文影响新连接</li>
</ul>

<h3>十一、TCP拥塞控制</h3>

<h4>1. 拥塞原因</h4>

<ul>
  <li>网络负载过大</li>
</ul>

<h4>2. 四种算法</h4>

<table border="1">
<tr>
<th>算法</th>
<th>作用</th>
</tr>

<tr>
<td>慢开始</td>
<td>指数增长</td>
</tr>

<tr>
<td>拥塞避免</td>
<td>线性增长</td>
</tr>

<tr>
<td>快重传</td>
<td>快速发现丢包</td>
</tr>

<tr>
<td>快恢复</td>
<td>恢复发送</td>
</tr>
</table>

<h4>3. 核心规则 </h4>

<ul>
  <li>CongWin &lt; Threshold：
    <ul>
      <li>慢开始（指数增长）</li>
    </ul>
  </li>

  <li>CongWin ≥ Threshold：
    <ul>
      <li>拥塞避免（线性增长）</li>
    </ul>
  </li>

  <li>超时：
    <ul>
      <li>Threshold = CongWin / 2</li>
      <li>CongWin = 1 MSS</li>
    </ul>
  </li>
</ul>
