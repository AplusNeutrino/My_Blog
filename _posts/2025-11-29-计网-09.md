<h2>第9章 互联网上的音频/视频服务</h2>

<h3>一、多媒体信息</h3>

<h4>定义</h4>
<ul>
  <li>文字、图像、音频、视频等综合信息</li>
</ul>

<h4>特点</h4>
<ul>
  <li>数据量大</li>
  <li>实时性要求高</li>
  <li>具有“边传输、边播放”特点</li>
</ul>

<hr>

<h3>二、时延抖动（Jitter）</h3>

<h4>定义</h4>
<ul>
  <li>分组到达时间不均匀</li>
</ul>

<h4>产生原因</h4>
<ul>
  <li>网络拥塞</li>
  <li>排队延迟变化</li>
</ul>

<h4>解决方法</h4>
<ul>
  <li>接收端缓存</li>
</ul>

<h4>考试陷阱</h4>
<ul>
  <li>缓存能减小抖动</li>
  <li>但会增加时延</li>
</ul>

<hr>

<h3>三、互联网音频/视频服务分类</h3>

<table border="1">
<tr>
<th>类型</th>
<th>特点</th>
</tr>

<tr>
<td>流式存储音频/视频</td>
<td>边下载边播放</td>
</tr>

<tr>
<td>流式实况音频/视频</td>
<td>边录制边发送</td>
</tr>

<tr>
<td>交互式音频/视频</td>
<td>实时双向通信</td>
</tr>
</table>

<hr>

<h3>四、流式存储音频/视频</h3>

<h4>特点</h4>
<ul>
  <li>用户不必等全部下载完成</li>
  <li>可边接收边播放</li>
</ul>

<h4>元文件（Meta File）</h4>
<ul>
  <li>保存媒体文件信息</li>
  <li>包含 URL</li>
</ul>

<hr>

<h3>五、服务器类型</h3>

<table border="1">
<tr>
<th>服务器</th>
<th>作用</th>
</tr>

<tr>
<td>Web服务器</td>
<td>提供网页</td>
</tr>

<tr>
<td>媒体服务器</td>
<td>提供音视频流</td>
</tr>
</table>

<hr>

<h3>六、两种媒体传送方式</h3>

<table border="1">
<tr>
<th>方式</th>
<th>特点</th>
</tr>

<tr>
<td>HTTP</td>
<td>普通Web方式</td>
</tr>

<tr>
<td>专用流媒体协议</td>
<td>支持实时控制</td>
</tr>
</table>

<hr>

<h3>七、RTSP 协议</h3>

<h4>全称</h4>
<ul>
  <li>Real-Time Streaming Protocol</li>
</ul>

<h4>作用</h4>
<ul>
  <li>控制媒体播放</li>
</ul>

<h4>RTSP 功能</h4>
<ul>
  <li>播放</li>
  <li>暂停</li>
  <li>快进</li>
  <li>后退</li>
</ul>

<h4>重要特点</h4>
<ul>
  <li>RTSP 只控制</li>
  <li>不传送媒体数据</li>
</ul>

<h4>传输方式</h4>
<ul>
  <li>TCP</li>
  <li>UDP</li>
</ul>

<hr>

<h3>八、IP 电话（VoIP）</h3>

<h4>定义</h4>
<ul>
  <li>在 IP 网络上传输语音</li>
</ul>

<h4>别名</h4>
<ul>
  <li>VoIP</li>
  <li>Internet Telephony</li>
</ul>

<hr>

<h3>九、IP 电话网关</h3>

<h4>作用</h4>
<ul>
  <li>连接传统电话网与 IP 网络</li>
</ul>

<h4>主要功能</h4>
<ul>
  <li>语音格式转换</li>
  <li>呼叫建立</li>
  <li>地址转换</li>
</ul>

<hr>

<h3>十、IP 电话需要的协议</h3>

<table border="1">
<tr>
<th>协议类型</th>
<th>作用</th>
</tr>

<tr>
<td>信令协议</td>
<td>建立呼叫</td>
</tr>

<tr>
<td>媒体传输协议</td>
<td>传送语音数据</td>
</tr>
</table>

<hr>

<h3>十一、RTP 协议</h3>

<h4>全称</h4>
<ul>
  <li>Real-time Transport Protocol</li>
</ul>

<h4>作用</h4>
<ul>
  <li>实时数据传输</li>
</ul>

<h4>特点</h4>
<ul>
  <li>运行在 UDP 上</li>
  <li>不保证 QoS</li>
</ul>

<h4>RTP 提供</h4>
<ul>
  <li>时间戳</li>
  <li>序号</li>
</ul>

<h4>作用</h4>
<ul>
  <li>恢复播放顺序</li>
  <li>同步媒体</li>
</ul>

<hr>

<h3>十二、RTCP 协议</h3>

<h4>全称</h4>
<ul>
  <li>RTP Control Protocol</li>
</ul>

<h4>作用</h4>
<ul>
  <li>配合 RTP 工作</li>
</ul>

<h4>主要功能</h4>
<ul>
  <li>服务质量反馈</li>
  <li>统计信息</li>
  <li>同步控制</li>
</ul>

<h4>考试陷阱</h4>
<ul>
  <li>RTCP 不传输媒体数据</li>
</ul>

<hr>

<h3>十三、H.323</h3>

<h4>定义</h4>
<ul>
  <li>ITU-T 制定的视频会议标准</li>
</ul>

<h4>作用</h4>
<ul>
  <li>IP 电话与视频会议</li>
</ul>

<h4>H.323 四大组件</h4>

<table border="1">
<tr>
<th>组件</th>
<th>作用</th>
</tr>

<tr>
<td>H.323终端</td>
<td>用户设备</td>
</tr>

<tr>
<td>网关 Gateway</td>
<td>连接不同网络</td>
</tr>

<tr>
<td>网守 Gatekeeper</td>
<td>地址管理与控制</td>
</tr>

<tr>
<td>MCU</td>
<td>多点会议控制</td>
</tr>
</table>

<hr>

<h3>十四、SIP 协议</h3>

<h4>全称</h4>
<ul>
  <li>Session Initiation Protocol</li>
</ul>

<h4>作用</h4>
<ul>
  <li>建立会话</li>
</ul>

<h4>特点</h4>
<ul>
  <li>基于文本</li>
  <li>类似 HTTP</li>
</ul>

<h4>组成</h4>

<table border="1">
<tr>
<th>组成</th>
<th>作用</th>
</tr>

<tr>
<td>用户代理 UA</td>
<td>发起/接收呼叫</td>
</tr>

<tr>
<td>代理服务器</td>
<td>转发请求</td>
</tr>

<tr>
<td>重定向服务器</td>
<td>返回目标地址</td>
</tr>
</table>

<hr>

<h3>十五、SIP URI</h3>

<h4>格式</h4>
<pre>sip:username@domain</pre>

<h4>作用</h4>
<ul>
  <li>标识 SIP 用户</li>
</ul>

<hr>

<h3>十六、SDP 协议</h3>

<h4>全称</h4>
<ul>
  <li>Session Description Protocol</li>
</ul>

<h4>作用</h4>
<ul>
  <li>描述媒体会话信息</li>
</ul>

<hr>

<h3>十七、QoS（服务质量）</h3>

<h4>全称</h4>
<ul>
  <li>Quality of Service</li>
</ul>

<h4>作用</h4>
<ul>
  <li>保证网络服务质量</li>
</ul>

<h4>QoS指标</h4>
<ul>
  <li>带宽</li>
  <li>时延</li>
  <li>抖动</li>
  <li>丢包率</li>
</ul>

<hr>

<h3>十八、调度与管制机制</h3>

<h4>1. FIFO（先进先出）</h4>
<ul>
  <li>先到先服务</li>
</ul>

<h4>缺点</h4>
<ul>
  <li>不区分优先级</li>
</ul>

<hr>

<h4>2. 优先级排队</h4>
<ul>
  <li>高优先级先发送</li>
</ul>

<hr>

<h4>3. 公平排队 FQ</h4>
<ul>
  <li>各流公平共享带宽</li>
</ul>

<hr>

<h4>4. 加权公平排队 WFQ</h4>
<ul>
  <li>按权重分配带宽</li>
</ul>

<h4>考试高频</h4>
<ul>
  <li>WFQ 比 FIFO 更公平</li>
</ul>

<hr>

<h3>十九、管制机制（Traffic Policing）</h3>

<h4>控制参数</h4>
<ul>
  <li>平均速率</li>
  <li>峰值速率</li>
  <li>突发长度</li>
</ul>

<hr>

<h3>二十、IntServ 综合服务</h3>

<h4>全称</h4>
<ul>
  <li>Integrated Services</li>
</ul>

<h4>特点</h4>
<ul>
  <li>为单个流保留资源</li>
</ul>

<h4>关键协议</h4>
<ul>
  <li>RSVP</li>
</ul>

<hr>

<h3>二十一、RSVP 协议</h3>

<h4>作用</h4>
<ul>
  <li>资源预留</li>
</ul>

<h4>特点</h4>
<ul>
  <li>网络层控制协议</li>
  <li>不携带应用数据</li>
</ul>

<hr>

<h3>二十二、DiffServ 区分服务</h3>

<h4>全称</h4>
<ul>
  <li>Differentiated Services</li>
</ul>

<h4>思想</h4>
<ul>
  <li>对不同数据流提供不同等级服务</li>
</ul>

<h4>边界路由器功能</h4>
<ul>
  <li>分类</li>
  <li>标记</li>
  <li>整形</li>
</ul>

<h4>DS 字段</h4>
<ul>
  <li>标记服务等级</li>
</ul>

<hr>

<h3>二十三、PHB</h3>

<h4>全称</h4>
<ul>
  <li>Per-Hop Behavior</li>
</ul>
