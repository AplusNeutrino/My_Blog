<h2>第10章 下一代因特网</h2>

<h3>一、IPv6 概述</h3>

<h4>IPv6 出现原因</h4>
<ul>
  <li>IPv4 地址耗尽</li>
  <li>互联网快速发展</li>
</ul>

<h4>IPv6 特点</h4>
<ul>
  <li>地址空间巨大</li>
  <li>提高路由效率</li>
  <li>支持自动配置</li>
  <li>支持QoS</li>
  <li>增强安全性</li>
</ul>

<hr>

<h3>二、IPv6 地址</h3>

<h4>长度</h4>
<ul>
  <li>128 位</li>
</ul>

<h4>表示方法</h4>
<ul>
  <li>采用十六进制</li>
  <li>每16位一组</li>
  <li>组间用冒号分隔</li>
</ul>

<h4>示例</h4>
<pre>
ABCD:EF01:0000:0000:0000:0000:1234:5678
</pre>

<hr>

<h3>三、IPv6 地址压缩规则</h3>

<h4>规则1</h4>
<ul>
  <li>前导 0 可以省略</li>
</ul>

<h4>例子</h4>
<pre>
0ABC → ABC
</pre>

<hr>

<h4>规则2</h4>
<ul>
  <li>连续多个 0 可压缩为 ::</li>
</ul>

<h4>例子</h4>
<pre>
FF05:0:0:0:0:0:0:B3
→ FF05::B3
</pre>

<h4>考试陷阱</h4>
<ul>
  <li>:: 在一个地址中只能出现一次</li>
</ul>

<hr>

<h3>四、CIDR 表示法</h3>

<h4>作用</h4>
<ul>
  <li>表示网络前缀长度</li>
</ul>

<h4>示例</h4>
<pre>
12AB::CD30:0:0:0/60
</pre>

<hr>

<h4>基本首部长度</h4>

<hr>

<h3>九、IPv4 向 IPv6 过渡技术</h3>

<h4>1. 双协议栈</h4>

<ul>
  <li>同时运行 IPv4 与 IPv6</li>
</ul>

<hr>

<h4>2. 隧道技术</h4>

<ul>
  <li>IPv6 数据封装在 IPv4 中传输</li>
</ul>

<hr>

<h3>十、MPLS（多协议标记交换）</h3>

<h4>全称</h4>
<ul>
  <li>Multiprotocol Label Switching</li>
</ul>

<h4>作用</h4>
<ul>
  <li>提高转发效率</li>
</ul>

<h4>特点（必考）</h4>
<ul>
  <li>不是替代 IP</li>
  <li>属于增强技术</li>
</ul>

<hr>

<h3>十一、MPLS 的三大特点</h3>

<ul>
  <li>支持QoS</li>
  <li>支持流量工程</li>
  <li>支持VPN</li>
</ul>

<hr>

<h3>十二、MPLS 工作原理</h3>

<h4>核心思想</h4>
<ul>
  <li>给分组添加“标记 Label”</li>
  <li>按标记转发</li>
</ul>

<h4>考试高频</h4>
<ul>
  <li>MPLS 转发速度比传统查路由表更快</li>
</ul>

<hr>

<h3>十三、MPLS 域</h3>

<h4>定义</h4>
<ul>
  <li>由支持 MPLS 的路由器组成</li>
</ul>

<hr>

<h3>十四、LSR</h3>

<h4>全称</h4>
<ul>
  <li>Label Switching Router</li>
</ul>

<h4>作用</h4>
<ul>
  <li>根据标记交换分组</li>
</ul>

<hr>

<h3>十五、FEC 转发等价类</h3>

<h4>全称</h4>
<ul>
  <li>Forwarding Equivalence Class</li>
</ul>

<h4>定义</h4>
<ul>
  <li>采用相同方式转发的一组分组</li>
</ul>

<h4>特点</h4>
<ul>
  <li>同一 FEC 使用同一标记</li>
</ul>

<hr>

<h3>十六、P2P 应用</h3>

<h4>定义</h4>
<ul>
  <li>对等节点之间直接通信</li>
</ul>

<h4>特点</h4>
<ul>
  <li>去中心化</li>
  <li>节点地位平等</li>
  <li>资源共享</li>
</ul>

<hr>

<h3>十七、P2P 节点特点</h3>

<ul>
  <li>既可作客户端</li>
  <li>也可作服务器</li>
</ul>

<hr>

<h3>十八、P2P 的两类</h3>

<th>特点</th>
</tr>

<tr>

</tr>
</table>

<hr>

<h3>十九、Napster</h3>

<h4>特点</h4>
<ul>
  <li>集中式 P2P</li>
</ul>

<h4>工作方式</h4>
<ul>
  <li>服务器负责文件索引</li>
  <li>用户之间直接传输文件</li>
</ul>

<hr>

<h3>二十、Gnutella</h3>

<h4>特点</h4>
<ul>
  <li>全分布式 P2P</li>
</ul>

<hr>

<h3>二十一、BitTorrent（BT）</h3>

<h4>特点</h4>
<ul>
  <li>文件分块下载</li>
  <li>边下载边上传</li>
</ul>
